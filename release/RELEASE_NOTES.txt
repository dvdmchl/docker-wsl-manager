# Docker WSL Manager - Release 1.2.0

## What's New in 1.2.0

### Features & Enhancements
- **Windows MSI Installer**: Added support for generating a standard Windows Installer (.msi) bundled with a custom runtime.
- **Enhanced Log Viewer**: Refactored to support real-time log streaming with ANSI color support.
- **Smart Scroll Lock**: Implemented a robust scroll-lock mechanism that pauses auto-scrolling while reading history and resumes at the bottom.
- **Container Details Page**: Dedicated space to view output and control the container.
- **Reactive UI**: Action buttons (Start, Stop, Restart) now automatically react to the container's running state.
- **Dynamic Keyboard Shortcuts**: Configurable shortcuts (e.g., Ctrl+S, Ctrl+R) via the Settings menu.
- **Auto-Update Check**: Automatically checks for newer versions on GitHub during startup.

### Fixes & Quality
- **SonarQube Quality Gate**: Resolved multiple BLOCKER, CRITICAL, and MAJOR findings, significantly improving code reliability and security.
- **Redundant Message Suppression**: Fixed issues with duplicate alert popups during network errors.
- **Improved Stability**: Resolved multiple potential memory leaks and null pointer issues.
- **Stream Reliability**: Fixed an issue where logs would stop updating after a container restart.

## Installation

### MSI Installer (Recommended for Windows)
1. Download `DockerWSLManager-1.2.0.msi`
2. Run the installer and follow instructions
3. Launch from Start Menu or Desktop shortcut

### Standalone JAR
1. Ensure Java 22 or higher is installed
2. Run the application: `java -jar docker-wsl-manager.jar`
